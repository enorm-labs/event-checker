## Field Mapping Analysis

The data model was designed by analyzing the following Berlin venue websites. Each section documents which website elements
map to which [data model](DATA_MODEL.md) fields.

### Astra Kulturhaus

Source: [astra-berlin.de](https://www.astra-berlin.de/)

| Website Element                   | Mapped To                         |
|-----------------------------------|-----------------------------------|
| Event title                       | `event.title`                     |
| Event kind (Concert, Festival, …) | `event.event_type`                |
| Date                              | `event.event_date`                |
| Doors time                        | `event.doors_time`                |
| Start time                        | `event.start_time`                |
| Subtitle / tour name              | `event.subtitle`                  |
| Description / artist bio          | `event.description`               |
| Poster image                      | `event.image_url`                 |
| "sold out" badge                  | `event.sold_out`                  |
| Event URL slug                    | `event.slug` / `event.source_url` |
| Venue name & address              | `venue.*`                         |
| Presale price (Vorverkauf)        | `event.price_presale`             |
| Box office price (Abendkasse)     | `event.price_box_office`          |
| Promoter / presenter              | `promoter.*`                      |
| Artist name, bio, links           | `artist.*`                        |
| Artist role (headliner, support)  | `event_artist.role`               |
| Schema.org `MusicEvent`           | Confirms the field mapping        |

### Festsaal Kreuzberg

Source: [festsaal-kreuzberg.de](https://festsaal-kreuzberg.de/de)

| Website Element                          | Mapped To                          |
|------------------------------------------|------------------------------------|
| Event title                              | `event.title`                      |
| Subtitle / tour name                     | `event.subtitle`                   |
| Category / genre tag (Party, German Pop) | `event.event_type` / `event.genre` |
| Date (e.g. "Do, 14 Mai 2026")            | `event.event_date`                 |
| Einlass (doors)                          | `event.doors_time`                 |
| Beginn (start)                           | `event.start_time`                 |
| Description text                         | `event.description`                |
| Event image                              | `event.image_url`                  |
| Event page URL (`/de/programm/slug/`)    | `event.source_url`                 |
| External ticket link (eventim, etc.)     | `event.ticket_url`                 |
| Partner / promoter section               | `promoter.*`                       |

*Notes:*

- *Festsaal Kreuzberg is a Nuxt.js SPA backed by Wagtail CMS. Event data is server-side rendered in the HTML but heavily wrapped in CSS/SVG. No structured
  data (schema.org) or public API is available; the `ld+json` script tag is present but empty.*
- *The venue has multiple spaces/rooms (e.g. "Festsaal", "Festsaal22"). An `event-location` field identifies which room, but this is venue-internal and does not
  require a schema change — the venue entity already represents Festsaal Kreuzberg as a whole.*
- *No prices are shown on the event pages; pricing is only available via the external ticket link (e.g. eventim.de). Similar to Bi Nuu.*
- *No sold-out or cancelled/postponed badges were observed, but the template likely supports them.*
- *No Facebook event URLs or artist social links are exposed.*
- *Some events embed Spotify/YouTube iframes via an `ItemIframe` component; these are not importable data.*

### Fluxbau

Source: [fluxfm.de/fluxbau](https://www.fluxfm.de/fluxbau)

| Website Element                         | Mapped To                     |
|-----------------------------------------|-------------------------------|
| Artist name                             | `event.title` / `artist.name` |
| Artist Instagram link                   | `artist.instagram_url`        |
| Date (e.g. "14. Mai" — no year shown)   | `event.event_date`            |
| Doors time (Einlass)                    | `event.doors_time`            |
| External ticket link (eventim)          | `event.ticket_url`            |
| Recurring event name (e.g. "Quizlabor") | `event.title`                 |

*Notes:*

- *Fluxbau is operated by radio station 100.6 FluxFM. The event page is part of the FluxFM website, not a standalone venue site.*
- *Very sparse listings — typically only ~1 upcoming event shown at a time, plus recurring series.*
- *No individual event pages — all events are inline on a single page. `source_url` would point to the Fluxbau page itself.*
- *Dates omit the year (e.g. "14. Mai") — the importer would need to infer it from context.*
- *Recurring events use natural-language schedules (e.g. "Jeden 1. Dienstag im Monat") rather than listing individual dates.
  The importer would need to generate individual event instances or skip these.*
- *The site is an Angular SPA (radiosphere platform), so scraping may require a headless browser or API interception.*
- *No event descriptions, images, genres, prices, sold-out/cancelled badges, Facebook links, or promoter info are shown.*
- *No schema changes required — all available data maps to existing nullable fields.*

### Neue Zukunft

Source: [neue-zukunft.org](https://neue-zukunft.org/)

| Website Element                       | Mapped To                               |
|---------------------------------------|-----------------------------------------|
| Widget event `name`                   | `event.title`                           |
| Widget `start.date` / `start.time`    | `event.event_date` / `event.start_time` |
| Widget HTML `description`             | `event.description`                     |
| Widget `coverImage.url`               | `event.image_url`                       |
| Widget `actions[]` "Get Tickets" link | `event.ticket_url`                      |
| Widget `actions[]` "Sold Out!" marker | `event.sold_out`                        |

*Notes:*

- *Neue Zukunft is a multi-purpose cultural venue (cinema, concerts, theater, bar). The public site is a static landing page with links to sub-projects, and the
  concert programme is also published as an image-based monthly **PDF flyer** (e.g. `Konzerte-05-2026-Neue-Zukunft.pdf`) — not machine-readable.*
- *However, the landing page embeds an **Elfsight "Event Calendar" widget** whose public JSON boot API (`core.service.elfsight.com/p/boot/?w=<widgetId>`)
  returns every concert as clean structured data — the stable import source used by `NeueZukunftWebsiteImporter` (no headless browser or OCR needed).*
- *No per-event URLs (the widget is anchor-based), so `source_url` points at the landing page; `sourceId` uses the widget's stable event UUID. No event-category
  field — the venue is live-music, so events default to `CONCERT` (festival titles → `FESTIVAL`).*
- *Events are also announced via Instagram (`@neuezukunftstralau`), which is not a practical import source. No schema changes required.*

## Data Model Completeness Summary

After analyzing all venue and promoter websites listed above, **no schema changes are required**. The existing
data model comprehensively covers all data fields observed across all sources. Key observations:

- All event data (titles, dates, times, prices, genres, statuses) maps cleanly to existing `event` columns.
- Artist social links (Instagram, Bandcamp, YouTube) map to existing `artist` columns.
- Promoter data is consistently available and maps to the `promoter` entity.
- Event statuses (cancelled, postponed, relocated, sold out) are fully covered by `EventStatus` enum and `sold_out` flag.
- Non-standard pricing ("donation", "Eintritt frei", conditional pricing) is handled by `price_note`.
- Sub-venues / multiple floors are venue-internal details that don't require schema changes.
